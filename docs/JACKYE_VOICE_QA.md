# Jackye Voice QA Check (use before you ship anything)

This is not theoretical. **This is pass / fail.**

Run every piece of content, UI copy, or output through this.

**Canonical stack:** Use alongside [`BUILD_BIBLE.md`](BUILD_BIBLE.md), [`MESSAGE_SYSTEM.md`](MESSAGE_SYSTEM.md), [`../newsletter/style/jackye-voice.md`](../newsletter/style/jackye-voice.md), and [`../newsletter/style/enforcement.md`](../newsletter/style/enforcement.md).

---

## 1. The reality check

**Does this sound like someone who has actually seen this before?**

If it sounds like:

- general advice  
- surface-level commentary  
- something anyone could say  

→ **FAIL**

You should feel: **this came from experience, not observation.**

---

## 2. The no-fluff test

**Remove 30% of the words. Does it get better?**

- If yes → **FAIL** (you had fluff)  
- If no → **PASS** (you’re clean)

---

## 3. The signal test

**Is there a real signal or pattern in this?**

If it’s just:

- summary  
- commentary  
- vibes  

→ **FAIL**

Every piece should answer: **what is actually happening here?**

---

## 4. The “so what” test

**Does it clearly tell me what this means?**

If the reader has to think: *okay… but what does this mean for me?* → **FAIL**

You need: **interpretation, not description.**

---

## 5. The decision test

**Does it help me decide something?**

If not → **FAIL**

Every piece should move toward: **act · pause · question · verify**

---

## 6. The human test

**Does this sound like something you would actually say out loud?**

If it sounds like:

- a keynote  
- a blog post  
- a corporate memo  

→ **FAIL**

You want: **“I’ve heard her say something like this.”**

---

## 7. The edge test

**Is there at least one line that hits?**

If everything is “good” but nothing stands out → **FAIL**

You need one moment where the reader goes: **“oh.”**

---

## 8. The anti-LinkedIn test

**Would this get lost in a feed?**

If yes → **FAIL**

If it sounds like *“in today’s world…”* or *“it’s important to…”* — **kill it.**

---

## 9. The receipts test

**Could you defend this if someone asked “how do you know?”**

If not → **FAIL**

No made-up confidence.

---

## 10. The trust test (most important)

**Does this build trust or perform intelligence?**

If it feels like:

- trying to sound smart  
- trying to impress  

→ **FAIL**

If it feels like: **“this is useful, I trust this”** → **PASS**

---

## Quick scoring version (when you’re moving fast)

Ask:

1. Is this **real**?  
2. Is this **clear**?  
3. Does this help someone **decide**?  
4. Does this sound like **me**?

If any answer is **no** → **don’t ship it.**

---

## Final filter

Before anything goes live, ask:

**Would I stand behind this if my name was the only thing on it?**

If you hesitate → **don’t ship it.**
